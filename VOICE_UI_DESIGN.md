## 🔄 互動流程

### 流程1：自動播放模式
```
1. 問題顯示
   ↓
2. 系統自動朗讀法文（延遲 0.5 秒）
   ↓
3. 用戶閱讀和聆聽
   ↓
4. 用戶選擇答案
   ↓
5. 顯示反饋
   ↓
6. 系統自動朗讀正確答案（延遲 0.8 秒）
```

### 流程2：手動播放模式
```
1. 問題顯示（不自動播放）
   ↓
2. 用戶主動點擊🔊按鈕
   ↓
3. 系統朗讀對應的法文
   ↓
4. 用戶可重複點擊聆聽
   ↓
5. 用戶選擇答案
   ↓
6. 反饋中可再次點擊播放
```

## 📱 觸控裝置優化

### 手機/平板特殊處理
- 按鈕最小觸控區域：44x44 像素
- 語音按鈕放大到 40x40 像素
- 增加按鈕間距（防止誤觸）
- 優化滑桿觸控體驗

### 手勢支援
```
單擊 🔊 按鈕 = 播放
雙擊 🔊 按鈕 = 重新播放
長按 = 顯示更多選項（未來功能）
```

## 🎯 無障礙設計

### 鍵盤操作
```
Tab        = 切換焦點
Space/Enter = 啟動按鈕
←/→        = 調整語速滑桿
Esc        = 停止播放
```

### 螢幕閱讀器
```html
<button aria-label="播放法文發音：Bonjour">
  🔊
</button>

<button aria-label="自動播放已開啟，點擊關閉">
  🔊 自動播放：開
</button>
```

## 💡 設計理念

1. **直覺性**：🔊 圖示全球通用，一看就懂
2. **即時性**：點擊立即播放，無需等待
3. **可控性**：自動/手動隨時切換
4. **視覺性**：播放中有明確的視覺反饋
5. **無障礙**：支援鍵盤、螢幕閱讀器
